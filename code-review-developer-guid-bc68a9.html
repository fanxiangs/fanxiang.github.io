Introduction

这次我们来讨论一下如何才能做好 code review，算是对 Code Review Developer Guide 进行总结。code review 的过程中，有两个重要的角色 reviewer 和 commiter。

Guidance for code reviewers

Code Reviewer 的步骤

  1. 全面了解改动了哪些东西

    通过查看 commit message 来了解这次改动是为了什么，并检查commit msg 是否符合规范，相应的字段填写是否正确，比如RFC、Description、Test

  2. 先主后次，先整体后详细

    先看主要改动文件，再看改动较少文件

  3. 确保不漏下一个文件

Code Reviewer 的要点

In doing a code review, you should make sure that:

  • The code is well-designed.
  • The functionality is good for the users of the code.
  • Any UI changes are sensible and look good.
  • Any parallel programming is done safely.
  • The code isn’t more complex than it needs to be.
  • The developer isn’t implementing things they might need in the future but don’t know they need now.
  • Code has appropriate unit tests.（可能本就没有unittest）
  • Tests are well-designed.（有Tests就已经很棒了）
  • The developer used clear names for everything.（起名字总是一件难事情）
  • Comments are clear and useful, and mostly explain why instead of what.（如何写注释一个好问题）
  • Code is appropriately documented (generally in g3doc).
  • The code conforms to our style guides.

综上所述，乞丐版：

  • 首先检查code style：基础的空格、换行问题
  • 是否引入高的复杂度：毕竟日后可能落到自己手上，看不懂就要求写注释，好的代码必须是简单简洁的
